AI/Human Text Detector

Detectra is an advanced AI-powered text analysis tool that determines whether a piece of text was written by a human or generated by artificial intelligence. In an age where AI-generated content is becoming increasingly sophisticated, our tool provides a reliable method to distinguish between human and machine authorship.

Our algorithm analyzes multiple linguistic dimensions including semantic coherence, syntactic patterns, and stylistic features to provide accurate detection results.

How Our Detector Works

  • Pattern Recognition: Identifies repetitive structures common in AI-generated text
  • Semantic Analysis: Evaluates the logical flow and coherence of ideas
  • Stylometric Analysis: Examines writing style, vocabulary diversity, and sentence complexity
  • Statistical Modeling: Uses machine learning models trained on millions of text samples

How Detectra Works

Detectra uses a sophisticated pipeline to analyze text and determine its origin:

1

Data Collection

Combined 3 datasets (HuggingFace, Kaggle, HC) totaling 140,147 text samples

2

Text Preprocessing

Cleaned and tokenized text using spaCy with disabled components for efficiency

3

Feature Extraction

TF-IDF with (1,3) n-grams, 20,000 features, and stylometric analysis

4

Model Training

Trained multiple ML models and selected best performing LinearSVM

Key Finding: Our experiments showed that using ngram_range=(1,3), max_features=20000, and stop_words=None yielded the best results (91.8% accuracy with Logistic Regression). LinearSVM achieved 92.5% accuracy with balanced precision and recall across both classes.

About This Project

This AI/Human Text Detection project was developed as a comprehensive machine learning solution to address the growing challenge of distinguishing between human-written and AI-generated text.

Project Journey

  • Environment Setup: Python virtual environment with all required packages
  • Data Collection: Combined multiple authentic datasets (HuggingFace, Kaggle, HC)
  • Preprocessing: Efficient text cleaning and tokenization using spaCy
  • Model Development: Built and tested 4 different machine learning models
  • Optimization: Experimented with different feature configurations
  • Evaluation: Achieved 92.5% accuracy with LinearSVM model

Technical Challenges Overcome

Based on your experiments:

  • Vocabulary Size: Found that 30,000 features reduced accuracy to 90.2% (worse than 20,000 features)
  • Stopword Removal: Discovered that keeping stopwords improved accuracy compared to removing them
  • N-gram Range: (1,3) n-grams performed better than (1,2) for most configurations
  • Model Selection: LinearSVM outperformed other classifiers for this task
